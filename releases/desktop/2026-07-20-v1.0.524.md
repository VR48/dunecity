# DuneCity 1.0.524 Desktop

This hotfix repairs deployment of the bundled Tornie mod from the desktop
package into the user's mod directory.

## Fixed

- Install the complete Tornie payload, including campaigns, Tharpique custom-
  house registration, Mentat declarations, graphics, sounds, and integrity
  metadata.
- Preserve the bundled `mod.ini` instead of replacing it with incomplete
  generated metadata.
- Detect incomplete or outdated Tornie installations through required payload
  checks and `checksums.sha256`, allowing campaign-only updates to reseed.

## Compatibility

- Existing saves and the multiplayer protocol are unchanged.
- Vanilla, DuneCity, Dune2R, and Tornie remain separately selectable modes.
- Existing incomplete Tornie installations repair themselves on first launch.

## Verification

- Windows x64 Release build
- Complete test suite
- Repaired a reproduced four-file Tornie profile to the complete 700-file
  payload, including 207 campaign files.
- Confirmed a second launch does not unnecessarily reseed an unchanged payload.
