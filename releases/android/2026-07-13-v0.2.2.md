# Dune Legacy Android 0.2.2

Android ARM64 save-compatibility update based on DuneCity/Tornie `1.0.359`,
with the Dune2R mod scaffold and Android runtime adaptations layered on top.

## Fixed

- Campaign saves no longer fail with an end-of-file error while loading map
  tiles after the nine-slot team-array expansion.
- Tile exploration and last-access data now serialize exactly eight
  house-indexed slots, matching the packed save byte and preserving format
  `9818` compatibility.
- Object visibility loading no longer indexes past its temporary array and now
  preserves the eighth packed visibility bit.
- Loaded Dune City maps reconcile zone tiles, zone density, structure power
  draw, R/C/I population, valves, and simulation timing before play resumes.
- Save-load failures now log the active section, byte offset, and file length
  to make future compatibility problems directly diagnosable.

## Compatibility

- Architecture: `arm64-v8a`
- Minimum Android API: 23 (Android 6.0)
- Target Android API: 34
- Tested on Android 13 with a Ulefone Armor 21
- Installs in place and preserves existing app data and save files
- Verified by loading an existing Dune City campaign save and observing the
  city simulation continue advancing after load
